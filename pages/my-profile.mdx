## 프로젝트

### 외부 저장매체 반입 검사 중앙관리 시스템 고도화

- 담당 역할 : 백엔드 개발 1인
- 기술 스택 : **java, spring boot, elasticsearch, nginx, azure devops pipeline, docker**
- 운영 방식 : On-premise

### 프로젝트 소개

외부 저장매체 반입 시 악성코드를 포함하고 있는 외부 저장매체에 대한 사전 차단을 목표로 개발된 GateXcanner라는 제품의 중앙관리 시스템이 존재하였으나
오래전 MFC로 개발되었고 윈도우 환경만을 위한 설치형 애플리케이션이라는 한계점과 유지 보수가 되어있지 않아 많은 버그와 기능 미지원 등의 이슈가 꾸준히 발생하여
어디서든 웹으로 접속할 수 있는 웹 기반의 새로운 중앙관리 시스템 개발을 진행하였습니다.

### 개발 핵심 소개

- SPRING AOP 를 활용하여 공통 로직 간소화
    - **Spring AOP**를 활용하여 @Logging Annotation 이 달린 Controller에 대한 요청과 응답에 대한 로그를 하나의 로직에서 공통적으로 적재할 수 있도록 하였습니다
    - **Spring AOP**를 활용하여 @Notifiy Annotation 이 달린 Controller에 대한 응답이 실패에 대한 응답이면 관리자에게 "알림" 메세지를 보내도록 하였습니다.
    - 이외에도 **Spring AOP** 를 활용하여 Elasticsearch 자체에서 지원하지 않는 중복되는 Value 검증 기능 등 다양한 공통 로직을 개발하였습니다.

- Elasticsearch 를 활용한 빠른 로그 검색
    - 운영 로그, 감사 로그, 검사 로그 등 로그성 데이터등 대량 데이터에서 빠른 조회를 하기 위해 역색인 방식의 Elasticsearch 를 도입하였습니다.

    <Callout type="warning" emoji="⚠">
        **Elasticsearch 도입 시 발생한 문제들**
        
        Elasticsearch 도입시 해당 기술에 대한 지식이 충분히 학습되어 있지 않은 상태였으며, 이에 따라 관계형 데이터베이스에 적재해야 할 데이터들과 Elasticsearch에 적재해야 할 데이터의 명확한 분리가 되지 않았고
        그 결과 관계형 데이터들을 Elastcsearch에 적재하는 불상사가 발생하여 곤혹을 치른 경험이 존재합니다
        관계형 데이터베이스를 사용해야 할 데이터를 Elasticsearch에 저장했을 때 발생하는 문제점 들은 아래와 같습니다
        **join 미지원, 빈번한 수정이 발생하는 데이터에 대한 속도 저하, 트랜잭션 미지원, deep paging 불가 등**
    </Callout>

- 클라우드 제품만 CI/CD? NO! 온프라미스도 CI/CD 구축
    - 추가되는 고객사에 따라 다양한 커스텀 기능들이 추가, 빠르게 쏟아지는 고객 요구사항을 로컬 빌드만으로는 감당이 안 되는 상황이 발생하였습니다.
    이를 해결하고자 **Azure Devops** 의 **Build pipeline, Release pipeline** 기능을 적극 활용하였습니다.
    사내 온프라미스 제품 배포 시 빌드 결과물들을 저장하던 NAS와 연동하기 위하여 cloud 빌드 서버가 아닌 **azure self hosted build agent** 를 직접 사내에 구축하였으며 이에 **사내 NAS를 마운팅** 하여 빌드 시 사내 NAS에 접근할 수 있도록 구성하였습니다.
    Build pipeline 과 Release pipeline 의 역할을 구분하여
    **Build pipeline** 에서는 **PullRequest 검증, 실제 Build 및 버저닝 자동화** 를 진행했으며,
    **Release pipeline** 에서는 온프라미스기 제품이기 때문에 **빌드 결과물 사내 NAS 업로드, 테스트를 위한 도커 이미지 제작 및 도커 레지스트리에 이미지 업로드** 등을 진행하였습니다.
